# Singularity Quick Reference Guide

This guide helps you transition from Docker to Singularity for running on HPC clusters like Isambard.

## Docker vs Singularity Command Comparison

| Task | Docker | Singularity |
|------|--------|-------------|
| Build from definition | `docker build -t name .` | `singularity build --fakeroot name.sif name.def` |
| Run interactively | `docker run -it --gpus all name` | `singularity shell --nv name.sif` |
| Execute command | `docker run name python script.py` | `singularity exec name.sif python script.py` |
| With GPU | `docker run --gpus all name` | `singularity exec --nv name.sif` |
| Mount volume | `docker run -v /host:/container` | `singularity exec --bind /host:/container` |
| Set env var | `docker run -e VAR=value` | `singularity exec --env VAR=value` |
| View help | `docker run name --help` | `singularity run-help name.sif` |

## Common Workflows

### 1. Building the container

**On Isambard login node (recommended):**

```bash
# SSH to Isambard
ssh username@isambard.ac.uk

# Set temporary directory to avoid filling /tmp
export SINGULARITY_TMPDIR=/path/to/your/workspace/tmp
export SINGULARITY_CACHEDIR=/path/to/your/workspace/.singularity/cache
mkdir -p $SINGULARITY_TMPDIR $SINGULARITY_CACHEDIR

# Build the container (takes ~10-20 minutes)
singularity build --fakeroot mtep.sif mtep.def

# Verify it works
singularity exec mtep.sif python --version
singularity exec mtep.sif uv --version
```

**On your local machine:**

```bash
# Navigate to project root
cd /path/to/multilingual-teacher-eval

# Build the container
singularity build --fakeroot mtep.sif mtep.def

# Verify it works
singularity exec mtep.sif python --version
singularity exec mtep.sif uv --version
```

### 2. Transferring to Isambard

```bash
# Copy the .sif file to the cluster
scp mtep.sif username@isambard.ac.uk:~/containers/

# Or rsync for resumable transfers
rsync -avzP mtep.sif username@isambard.ac.uk:~/containers/
```

### 3. Running on Isambard

```bash
# SSH into Isambard
ssh username@isambard.ac.uk

# Navigate to your workspace
cd /path/to/your/workspace

# Test the container
srun -N 1 --gpus 1 --pty singularity shell --nv ~/containers/mtep.sif

# Inside the container, test your script
python scripts/get_intrinsic_metrics.py --help

# Run from outside the container
srun -N 1 --gpus 1 singularity exec --nv \
    --env HF_TOKEN=$HF_TOKEN \
    ~/containers/mtep.sif \
    python scripts/get_intrinsic_metrics.py --model-name meta-llama/Llama-3.2-1B-Instruct
```

### 4. Batch job submission

Create a SLURM script (e.g., `run_container.slurm`):

```bash
#!/bin/bash
#SBATCH --job-name=mtep-job
#SBATCH --nodes=1
#SBATCH --gpus=1
#SBATCH --time=04:00:00
#SBATCH --output=logs/%x-%j.out
#SBATCH --error=logs/%x-%j.err

mkdir -p logs

srun singularity exec --nv \
    --env HF_TOKEN=$HF_TOKEN \
    ~/containers/mtep.sif \
    python scripts/get_intrinsic_metrics.py \
    --model-name meta-llama/Llama-3.2-1B-Instruct \
    --output-dir /path/to/outputs
```

Submit with:
```bash
export HF_TOKEN=your_token_here
sbatch run_container.slurm
```

## Key Singularity Concepts

### Image Format
- **Docker**: Images stored in layers by daemon
- **Singularity**: Single `.sif` file (Singularity Image Format)
  - Self-contained
  - Immutable by default
  - Easy to copy/share

### Permissions
- **Docker**: Runs as root inside container
- **Singularity**: Runs as your user (safer for HPC)
  - You have same UID/GID inside and outside
  - Can't escalate privileges
  - Home directory mounted by default

### GPU Access
- **Docker**: `--gpus all` or `--gpus 0,1`
- **Singularity**: `--nv` (NVIDIA) or `--rocm` (AMD)
  - Automatically binds GPU drivers
  - Must be on a GPU node

### Bind Mounts
Singularity automatically mounts:
- `$HOME`
- `/tmp`
- `$PWD` (current directory)

For other paths, use `--bind`:
```bash
singularity exec --bind /data:/mnt/data --bind /scratch:/scratch mtep.sif command
```

## Troubleshooting

### "No space left on device" during build
Singularity uses `/tmp` by default, which can fill up on login nodes. Set a different cache location:
```bash
# Point to your workspace directory with more space
export SINGULARITY_TMPDIR=/path/to/your/workspace/tmp
export SINGULARITY_CACHEDIR=/path/to/your/workspace/.singularity/cache
mkdir -p $SINGULARITY_TMPDIR $SINGULARITY_CACHEDIR

singularity build --fakeroot mtep.sif mtep.def

# Clean up after build
rm -rf $SINGULARITY_TMPDIR/*
```

### Building on login node best practices
- **Use your project/work directory**: Login node `/tmp` is often small
- **Set cache directories**: Avoid filling shared `/tmp` space
- **Build during off-peak hours**: Considerate to other users
- **Monitor your build**: Keep an eye on disk usage with `df -h`
- **Clean up**: Remove temporary files after successful build

### Python package not found
Make sure you're in the `/app` directory:
```bash
singularity exec mtep.sif bash -c "cd /app && python scripts/your_script.py"
```

Or set the working directory:
```bash
singularity exec --pwd /app mtep.sif python scripts/your_script.py
```

### GPU not detected
- Ensure you're on a GPU node: `srun --gpus 1 --pty bash`
- Use `--nv` flag: `singularity exec --nv mtep.sif nvidia-smi`
- Check CUDA: `singularity exec --nv mtep.sif python -c "import torch; print(torch.cuda.is_available())"`

### Environment variables not set
Pass them explicitly:
```bash
singularity exec --env HF_TOKEN=$HF_TOKEN --env-file .env mtep.sif command
```
